# Architecture Decision Records (ADRs)

## Purpose
ADRs capture non-trivial architecture decisions in a durable, reviewable format.

## Conventions
1. File naming: `NNNN-short-kebab-title.md`
2. Numbering: zero-padded and strictly increasing
3. Status values: `PROPOSED`, `ACCEPTED`, `SUPERSEDED`, `DEPRECATED`
4. Each ADR must include context, decision, consequences, and alternatives
5. Major architecture changes require a linked ADR in the same PR
