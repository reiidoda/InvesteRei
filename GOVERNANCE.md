# Governance

## Maintainer Responsibilities
- Review and merge contributions aligned with project architecture and security requirements.
- Keep dependency, migration, and API compatibility decisions explicit.
- Enforce code of conduct and contribution standards.

## Decision Process
- Architectural changes should include an update in `docs/` and PR rationale.
- Security-sensitive changes require risk notes and validation evidence.
- Breaking API changes require versioning or migration path documentation.

## Release Discipline
- Prefer additive migrations.
- Tag releases from tested commits.
- Publish release notes summarizing behavior changes and operational impact.
